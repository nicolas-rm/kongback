## Base

Base path: `/api/stations`

Todos los endpoints requieren autenticacion y permisos `stations.*`. El borrado desactiva el registro cambiando `status` a `inactive`.

## Alcance por compania

Requiere header `X-Company-Id: <companyId>`. Todas las operaciones quedan limitadas a esa compania. Si el acceso del usuario tiene `scopeKey: "subCompanyId"`, `subCompanyId` y la estacion consultada tambien deben pertenecer a una de esas subcompanias.

## Endpoints

| Metodo   | Ruta                | Permiso              | Descripcion             |
| -------- | ------------------- | -------------------- | ----------------------- |
| `POST`   | `/api/stations`     | `stations.create`    | Crea una estacion.      |
| `GET`    | `/api/stations`     | `stations.read-list` | Lista estaciones.       |
| `GET`    | `/api/stations/:id` | `stations.read-one`  | Obtiene una estacion.   |
| `PATCH`  | `/api/stations/:id` | `stations.update`    | Actualiza una estacion. |
| `DELETE` | `/api/stations/:id` | `stations.delete`    | Desactiva una estacion. |

## Listado

`GET /api/stations`

Query params: `page`, `limit`, `search`, `all`, `status`, `subCompanyId`.

`search` busca por `stationNumber` y `name`.

Para columnas visibles, usa `subCompany.name`; conserva `subCompanyId` para filtros y acciones.

Response `200`: resultado paginado de `Station` con `subCompany` summary y sin `address`.

## Crear

`POST /api/stations`

```json
{
    "subCompanyId": "79d03f1d-3da4-478e-806e-75ef9d86c992",
    "stationNumber": "ST-001",
    "name": "Estacion Centro",
    "lat": 25.6866,
    "lon": -100.3161,
    "status": "active",
    "address": {
        "city": "Monterrey",
        "state": "Nuevo Leon",
        "country": "MX"
    }
}
```

Requeridos: `subCompanyId`, `stationNumber`, `name`. `subCompanyId` debe estar activo y pertenecer a la compania enviada en `X-Company-Id`.

Opcionales: `lat`, `lon`, `status`, `address`. `lat` acepta -90 a 90; `lon` acepta -180 a 180.

Response `201`: `Station` con `subCompany` summary y `address`.

## Obtener

`GET /api/stations/:id`

Path params: `id` UUID de la estacion.

Response `200`: `Station` con `subCompany` summary y `address`.

## Actualizar

`PATCH /api/stations/:id`

Acepta `stationNumber`, `name`, `lat`, `lon`, `status` y `address`, todos opcionales.

Response `200`: `Station` actualizada con `subCompany` summary y `address`.

## Desactivar

`DELETE /api/stations/:id`

Response `200`:

```json
{
    "id": "48c478d5-d9a6-4a8a-a98f-6cc0a3ea7d3f",
    "status": "inactive"
}
```

## Notas para frontend

Este modulo tambien aplica las reglas CRUD de `Frontend Guidelines`: crear con campos visibles/derivados, editar solo campos documentados en `Actualizar`, confirmar desactivacion y refrescar el listado despues de cada mutacion.

El front debe enviar `X-Company-Id`. `subCompanyId` debe venir de un select de subcompanias disponibles para la compania seleccionada y el alcance del usuario.

Los combustibles disponibles de una estacion no se capturan en el formulario base de estacion. En el detalle o edicion de estacion, agrega una seccion `Combustibles disponibles` que consuma `GET /api/station-fuels?stationId=<id>` y use `POST`, `PATCH` o `DELETE /api/station-fuels` para administrar la relacion. No debe existir una pagina principal separada para `station-fuels` en el menu del frontend.

Campos recomendados para formularios:

- Subcompania: `subCompanyId`.
- Numero de estacion: `stationNumber`.
- Nombre: `name`.
- Latitud: `lat`, opcional.
- Longitud: `lon`, opcional.
- Estado: `status`, con `active` como valor inicial.
- Direccion: grupo opcional basado en `address`.

En tablas usa `subCompany.name`; conserva `subCompanyId` solo para filtros y payloads. Si no se capturan coordenadas, omite `lat` y `lon`.

## Errores comunes

| Estado | Caso                                                                  |
| ------ | --------------------------------------------------------------------- |
| `400`  | Relacion invalida, coordenadas fuera de rango o parametros invalidos. |
| `401`  | Request sin autenticacion valida.                                     |
| `403`  | Usuario autenticado sin el permiso requerido.                         |
| `404`  | Estacion no encontrada.                                               |
| `409`  | Conflictos de unicidad segun constraints Prisma.                      |
