---
title: Companies API
description: Endpoints para gestionar companias del dominio operativo.
---

## Base

Base path: `/api/companies`

Todos los endpoints requieren autenticacion y permisos `companies.*`. El borrado desactiva el registro cambiando `status` a `inactive`.

## Modelo

```json
{
  "id": "0ed7e7e1-2b6d-45cf-94c7-79bf03bbaf02",
  "key": "COMPANY001",
  "externalId": "EXT-001",
  "name": "Acme Logistics",
  "tradeName": "Acme",
  "status": "active",
  "address": {
    "street": "Main St",
    "exteriorNumber": "100",
    "interiorNumber": null,
    "neighborhood": "Centro",
    "municipality": "Monterrey",
    "city": "Monterrey",
    "state": "Nuevo Leon",
    "country": "MX",
    "postalCode": "64000",
    "references": null
  }
}
```

`status`: `active`, `inactive`, `suspended`.

En listados, `address` no se devuelve para mantener el payload liviano. `address` si aparece en detalle, create y update.

## Endpoints

| Metodo | Ruta | Permiso | Descripcion |
| --- | --- | --- | --- |
| `POST` | `/api/companies` | `companies.create` | Crea una compania. |
| `GET` | `/api/companies` | `companies.read-list` | Lista companias. |
| `GET` | `/api/companies/:id` | `companies.read-one` | Obtiene una compania. |
| `PATCH` | `/api/companies/:id` | `companies.update` | Actualiza una compania. |
| `DELETE` | `/api/companies/:id` | `companies.delete` | Desactiva una compania. |

## Listado

Query params: `page`, `limit`, `search`, `all`, `status`.

`search` busca por `key`, `externalId`, `name` y `tradeName`.

```http
GET /api/companies?page=1&limit=20&search=acme&status=active
```

## Crear

```json
{
  "key": "COMPANY001",
  "externalId": "EXT-001",
  "name": "Acme Logistics",
  "tradeName": "Acme",
  "status": "active",
  "address": {
    "street": "Main St",
    "city": "Monterrey",
    "state": "Nuevo Leon",
    "country": "MX",
    "postalCode": "64000"
  }
}
```

Requeridos: `key`, `name`.

Opcionales: `externalId`, `tradeName`, `status`, `address`. Strings vacios opcionales se transforman a `null`.

## Actualizar

`PATCH /api/companies/:id`

Acepta los mismos campos que crear, todos opcionales.

## Desactivar

`DELETE /api/companies/:id`

```json
{
  "id": "0ed7e7e1-2b6d-45cf-94c7-79bf03bbaf02",
  "status": "inactive"
}
```

## Errores comunes

| Estado | Caso |
| --- | --- |
| `400` | Body, query o path params invalidos. |
| `401` | Request sin autenticacion valida. |
| `403` | Usuario autenticado sin el permiso requerido. |
| `404` | Compania no encontrada. |
| `409` | Conflictos de unicidad segun constraints Prisma. |
