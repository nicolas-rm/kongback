title: Companies API

## Base

Base path: `/api/companies`

Todos los endpoints requieren autenticacion y permisos `companies.*`. El borrado desactiva el registro cambiando `status` a `inactive`.

## Workspace

Requiere header `X-Organization-Id: <organizationId>`. Todas las operaciones quedan limitadas al workspace seleccionado. El backend valida que el usuario tenga acceso a esa organizacion y que este activa.

Header de compania: `X-Company-Id: <companyId>`. Es opcional para usuarios con acceso completo al workspace y obligatorio para usuarios restringidos por compania. Debe ser una compania activa dentro del workspace y las lecturas, actualizaciones y desactivaciones quedan limitadas a esa compania.

`POST /api/companies` crea una nueva compania dentro del workspace. Es una accion de alcance workspace: requiere usuario global o acceso completo al workspace. Un usuario restringido por compania no puede crear companias aunque tenga una compania seleccionada, porque el nuevo registro quedaria fuera de su alcance.

## Modelo

```json
{
    "id": "0ed7e7e1-2b6d-45cf-94c7-79bf03bbaf02",
    "organizationId": "2b4c918a-e7e0-4e99-b6c1-c2efed00b478",
    "key": "COMPANY001",
    "externalId": "EXT-001",
    "name": "Acme Logistics",
    "tradeName": "Acme",
    "status": "active",
    "address": {
        "street": "Main St",
        "exteriorNumber": "100",
        "interiorNumber": null,
        "neighborhood": "Centro",
        "municipality": "Monterrey",
        "city": "Monterrey",
        "state": "Nuevo Leon",
        "country": "MX",
        "postalCode": "64000",
        "references": null
    }
}
```

`status`: `active`, `inactive`, `suspended`.

En listados, `address` no se devuelve para mantener el payload liviano. `address` si aparece en detalle, create y update.

## Endpoints

| Metodo   | Ruta                 | Permiso               | Descripcion                        |
| -------- | -------------------- | --------------------- | ---------------------------------- |
| `POST`   | `/api/companies`     | `companies.create`    | Crea una compania en el workspace. |
| `GET`    | `/api/companies`     | `companies.read-list` | Lista companias.                   |
| `GET`    | `/api/companies/:id` | `companies.read-one`  | Obtiene una compania.              |
| `PATCH`  | `/api/companies/:id` | `companies.update`    | Actualiza una compania.            |
| `DELETE` | `/api/companies/:id` | `companies.delete`    | Desactiva una compania.            |

## Listado

`GET /api/companies`

Query params: `page`, `limit`, `search`, `all`, `status`.

`search` busca por `key`, `externalId`, `name` y `tradeName`.

```http
GET /api/companies?page=1&limit=20&search=acme&status=active
```

Response `200`: resultado paginado de `Company` sin `address`.

## Crear

`POST /api/companies`

```json
{
    "key": "COMPANY001",
    "externalId": "EXT-001",
    "name": "Acme Logistics",
    "tradeName": "Acme",
    "status": "active",
    "address": {
        "street": "Main St",
        "city": "Monterrey",
        "state": "Nuevo Leon",
        "country": "MX",
        "postalCode": "64000"
    }
}
```

Requeridos: `key`, `name`.

Opcionales: `externalId`, `tradeName`, `status`, `address`. Strings vacios opcionales se transforman a `null`.

Requiere acceso global o acceso completo al workspace. No crea la compania dentro de `X-Company-Id`; ese header solo aplica como filtro/alcance para usuarios que ya trabajan dentro de una compania existente.

Response `201`: `Company` con `address`.

## Obtener

`GET /api/companies/:id`

Path params: `id` UUID de la compania.

Response `200`: `Company` con `address`.

## Actualizar

`PATCH /api/companies/:id`

Acepta los mismos campos que crear, todos opcionales.

Response `200`: `Company` actualizada con `address`.

## Desactivar

`DELETE /api/companies/:id`

Response `200`:

```json
{
    "id": "0ed7e7e1-2b6d-45cf-94c7-79bf03bbaf02",
    "status": "inactive"
}
```

## Errores comunes

| Estado | Caso                                                                                        |
| ------ | ------------------------------------------------------------------------------------------- |
| `400`  | Body, query o path params invalidos.                                                        |
| `401`  | Request sin autenticacion valida.                                                           |
| `403`  | Usuario autenticado sin el permiso requerido o sin alcance suficiente para crear companias. |
| `404`  | Compania no encontrada.                                                                     |
| `409`  | Conflictos de unicidad segun constraints Prisma.                                            |
