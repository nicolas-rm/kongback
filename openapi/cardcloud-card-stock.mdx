title: Cardcloud Card Stock API
description: Endpoints para gestionar stock externo de tarjetas Cardcloud.
---

## Base

Base path: `/api/cardcloud-card-stock`

Todos los endpoints requieren autenticacion y permisos `cardcloud-card-stock.*`. El borrado desactiva el registro cambiando `providerStatus` a `inactive`.

## Alcance por compania

Requiere header `X-Company-Id: <companyId>`. Todas las operaciones quedan limitadas a esa compania. El stock queda asociado directamente a `companyId`; si `assignedCardId` se envia, debe pertenecer a una tarjeta activa de esa misma compania. Si el acceso del usuario tiene `scopeKey: "subCompanyId"`, solo puede operar stock asignado a tarjetas de esas subcompanias.

## Endpoints

| Metodo   | Ruta                            | Permiso                          | Descripcion            |
| -------- | ------------------------------- | -------------------------------- | ---------------------- |
| `POST`   | `/api/cardcloud-card-stock`     | `cardcloud-card-stock.create`    | Crea stock Cardcloud.  |
| `GET`    | `/api/cardcloud-card-stock`     | `cardcloud-card-stock.read-list` | Lista stock.           |
| `GET`    | `/api/cardcloud-card-stock/:id` | `cardcloud-card-stock.read-one`  | Obtiene un registro.   |
| `PATCH`  | `/api/cardcloud-card-stock/:id` | `cardcloud-card-stock.update`    | Actualiza un registro. |
| `DELETE` | `/api/cardcloud-card-stock/:id` | `cardcloud-card-stock.delete`    | Desactiva un registro. |

## Listado

`GET /api/cardcloud-card-stock`

Query params: `page`, `limit`, `search`, `all`, `status`, `assignedCardId`.

`status` filtra `providerStatus`. `search` busca por `externalId`, `maskedPan` y `clientId`.

Para columnas visibles, usa `assignedCard.externalId`; conserva `assignedCardId` para filtros y acciones.

Response `200`: resultado paginado de `CardcloudCardStock`.

## Crear

`POST /api/cardcloud-card-stock`

```json
{
    "externalId": "CC-00001",
    "assignedCardId": "ff2584aa-e2fc-4d75-b11c-61e447f3cbb0",
    "maskedPan": "**** **** **** 1234",
    "clientId": "CLIENT-001",
    "balance": 1200.5,
    "providerStatus": "active",
    "syncedAt": "2026-07-04"
}
```

Requerido: `externalId`.

Opcionales: `assignedCardId`, `maskedPan`, `clientId`, `balance`, `providerStatus`, `syncedAt`. Si `assignedCardId` se envia, debe apuntar a una tarjeta activa de la compania enviada en `X-Company-Id`.

Response `201`: `CardcloudCardStock`.

## Obtener

`GET /api/cardcloud-card-stock/:id`

Path params: `id` UUID del registro Cardcloud.

Response `200`: `CardcloudCardStock`.

## Actualizar

`PATCH /api/cardcloud-card-stock/:id`

Acepta `assignedCardId`, `maskedPan`, `clientId`, `balance`, `providerStatus`, `syncedAt`, todos opcionales. El registro actual debe pertenecer a la compania enviada en `X-Company-Id` y cualquier `assignedCardId` nuevo tambien debe pertenecer a esa compania.

Response `200`: `CardcloudCardStock` actualizado.

## Desactivar

`DELETE /api/cardcloud-card-stock/:id`

Response `200`:

```json
{
    "id": "413117ed-83f0-4d74-bf7b-1e26d3a9b9de",
    "status": "inactive"
}
```

## Errores comunes

| Estado | Caso                                                                        |
| ------ | --------------------------------------------------------------------------- |
| `400`  | Relacion invalida, balance negativo, fecha invalida o parametros invalidos. |
| `401`  | Request sin autenticacion valida.                                           |
| `403`  | Usuario autenticado sin el permiso requerido.                               |
| `404`  | Registro Cardcloud no encontrado.                                           |
| `409`  | Conflictos de unicidad segun constraints Prisma.                            |
