title: Documents API
description: Endpoints para subir, consultar, descargar, actualizar y eliminar documentos.
---

## Base

Base path: `/api/documents`

Todos los endpoints requieren autenticacion y permisos `documents.*`. `DELETE` aplica soft delete y tambien elimina el archivo del almacenamiento.

## Endpoints

| Metodo | Ruta | Permiso | Descripcion |
| --- | --- | --- | --- |
| `POST` | `/api/documents` | `documents.create` | Sube y registra un documento. |
| `GET` | `/api/documents` | `documents.read-list` | Lista documentos activos. |
| `GET` | `/api/documents/:id/download` | `documents.download` | Descarga el archivo. |
| `GET` | `/api/documents/:id` | `documents.read-one` | Obtiene metadata de un documento. |
| `PATCH` | `/api/documents/:id` | `documents.update` | Actualiza metadata editable. |
| `DELETE` | `/api/documents/:id` | `documents.delete` | Elimina logicamente el documento. |

## Listado

`GET /api/documents`

Query params: `page`, `limit`, `search`, `all`, `category`, `organizationId`, `entityType`, `entityId`.

`search` busca por `title`, `originalName` y `description`.

Response `200`: resultado paginado de `Document`.

## Crear

`POST /api/documents`

Content-Type: `multipart/form-data`.

Campos:

| Campo | Tipo | Requerido | Descripcion |
| --- | --- | --- | --- |
| `file` | `file` | Si | Archivo a subir. |
| `title` | `string` | Si | Titulo visible. |
| `description` | `string \| null` | No | Descripcion opcional. |
| `category` | `string \| null` | No | Categoria libre. |
| `organizationId` | `uuid \| null` | No | Organizacion activa asociada. |
| `entityType` | `string \| null` | No | Tipo de entidad relacionada. |
| `entityId` | `string \| null` | No | ID de entidad relacionada. |
| `scopeKey` | `string \| null` | No | Nombre de scope. |
| `scopeId` | `string \| null` | No | ID de scope. |

MIME permitidos por defecto: `application/pdf`, `image/jpeg`, `image/png`, `image/webp`, `text/plain`.

Tamano maximo por defecto: `20 MB`.

El backend valida que el contenido real coincida con el MIME declarado.

Response `201`: `Document`.

## Obtener metadata

`GET /api/documents/:id`

Path params: `id` UUID del documento.

Response `200`: `Document`.

## Actualizar

`PATCH /api/documents/:id`

```json
{
  "title": "Contrato actualizado",
  "description": "Nueva descripcion",
  "category": "contracts"
}
```

Solo permite actualizar `title`, `description` y `category`.

Response `200`: `Document` actualizado.

## Descargar

`GET /api/documents/:id/download`

Devuelve stream binario con:

| Header | Valor |
| --- | --- |
| `Content-Type` | MIME real del documento. |
| `Content-Disposition` | `attachment` con nombre original. |

## Eliminar

`DELETE /api/documents/:id`

Path params: `id` UUID del documento.

Response `200`:

```json
{
  "id": "8304b951-7f41-4653-8ac5-0c3b7a86521d",
  "deleted": true
}
```

## Errores comunes

| Estado | Caso |
| --- | --- |
| `400` | Archivo requerido, MIME no permitido, tamano excedido o relacion invalida. |
| `401` | Request sin autenticacion valida. |
| `403` | Usuario autenticado sin el permiso requerido. |
| `404` | Documento o archivo almacenado no encontrado. |
