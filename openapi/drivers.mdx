## Base

Base path: `/api/drivers`

Todos los endpoints requieren autenticacion y permisos `drivers.*`. El borrado desactiva el registro cambiando `status` a `inactive`.

## Workspace

Requiere header `X-Organization-Id: <organizationId>`. Todas las operaciones quedan limitadas al workspace seleccionado. `subCompanyId` debe pertenecer a una subcompania activa de esa organizacion.

Header opcional: `X-Company-Id: <companyId>`. Si se envia, debe ser una compania activa dentro del workspace y `subCompanyId`, el conductor consultado y sus relaciones deben pertenecer a esa compania.

## Endpoints

| Metodo   | Ruta               | Permiso             | Descripcion             |
| -------- | ------------------ | ------------------- | ----------------------- |
| `POST`   | `/api/drivers`     | `drivers.create`    | Crea un conductor.      |
| `GET`    | `/api/drivers`     | `drivers.read-list` | Lista conductores.      |
| `GET`    | `/api/drivers/:id` | `drivers.read-one`  | Obtiene un conductor.   |
| `PATCH`  | `/api/drivers/:id` | `drivers.update`    | Actualiza un conductor. |
| `DELETE` | `/api/drivers/:id` | `drivers.delete`    | Desactiva un conductor. |

## Listado

`GET /api/drivers`

Query params: `page`, `limit`, `search`, `all`, `status`, `subCompanyId`, `userId`.

`search` busca por `name` y `externalReference`.

Para columnas relacionadas, usa `subCompany.name` y `user.fullName`; conserva `subCompanyId` y `userId` para acciones y formularios.

Response `200`: resultado paginado de `Driver` con `subCompany` y `user` summaries, sin `address`.

## Crear

`POST /api/drivers`

```json
{
    "subCompanyId": "79d03f1d-3da4-478e-806e-75ef9d86c992",
    "userId": "9f5f0f4a-8f88-4f0b-9a18-6eb3f1e9b7a1",
    "name": "Maria Garcia",
    "externalReference": "DRV-001",
    "status": "active"
}
```

Requeridos: `subCompanyId`, `name`. `subCompanyId` debe estar activo en el workspace seleccionado y, si se envia `X-Company-Id`, debe pertenecer a esa compania.

Opcionales: `userId`, `externalReference`, `status`, `address`. Si `userId` se envia, debe ser un usuario activo.

Response `201`: `Driver` con `subCompany`, `user` y `address`.

## Obtener

`GET /api/drivers/:id`

Path params: `id` UUID del conductor.

Response `200`: `Driver` con `subCompany`, `user` y `address`.

## Actualizar

`PATCH /api/drivers/:id`

Acepta `userId`, `name`, `externalReference`, `status` y `address`, todos opcionales. `userId` puede enviarse como `null` para desvincular.

Response `200`: `Driver` actualizado con `subCompany`, `user` y `address`.

## Desactivar

`DELETE /api/drivers/:id`

Response `200`:

```json
{
    "id": "d77d1344-93d4-4dc2-923a-b241185692de",
    "status": "inactive"
}
```

## Errores comunes

| Estado | Caso                                             |
| ------ | ------------------------------------------------ |
| `400`  | Relacion invalida o parametros invalidos.        |
| `401`  | Request sin autenticacion valida.                |
| `403`  | Usuario autenticado sin el permiso requerido.    |
| `404`  | Conductor no encontrado.                         |
| `409`  | Conflictos de unicidad segun constraints Prisma. |
