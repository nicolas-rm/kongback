## Base

Base path: `/api/fuels`

Todos los endpoints requieren autenticacion y permisos `fuels.*`. El borrado desactiva el registro cambiando `status` a `inactive`.

El catalogo de combustibles es global. Los endpoints de lectura pueden usarse como catalogo de referencia desde modulos operativos. Crear, actualizar y desactivar combustibles requiere acceso global; no se ejecuta dentro de `X-Company-Id`.

## Modelo

```json
{
    "id": "e9ef8b38-c983-4b9c-a51c-7d39ff7b1c84",
    "code": "DIESEL",
    "name": "Diesel",
    "status": "active"
}
```

## Endpoints

| Metodo   | Ruta             | Permiso           | Descripcion               |
| -------- | ---------------- | ----------------- | ------------------------- |
| `POST`   | `/api/fuels`     | `fuels.create`    | Crea un combustible.      |
| `GET`    | `/api/fuels`     | `fuels.read-list` | Lista combustibles.       |
| `GET`    | `/api/fuels/:id` | `fuels.read-one`  | Obtiene un combustible.   |
| `PATCH`  | `/api/fuels/:id` | `fuels.update`    | Actualiza un combustible. |
| `DELETE` | `/api/fuels/:id` | `fuels.delete`    | Desactiva un combustible. |

## Listado

`GET /api/fuels`

Query params: `page`, `limit`, `search`, `all`, `status`.

`search` busca por `code` y `name`.

Response `200`: resultado paginado de `Fuel`.

## Crear

`POST /api/fuels`

```json
{
    "code": "diesel",
    "name": "Diesel",
    "status": "active"
}
```

Requeridos: `code`, `name`. `code` se normaliza a mayusculas.

Opcional: `status`.

Requiere acceso global.

Response `201`: `Fuel`.

## Obtener

`GET /api/fuels/:id`

Path params: `id` UUID del combustible.

Response `200`: `Fuel`.

## Actualizar

`PATCH /api/fuels/:id`

Acepta `code`, `name` y `status`, todos opcionales.

Requiere acceso global.

Response `200`: `Fuel` actualizado.

## Desactivar

`DELETE /api/fuels/:id`

Requiere acceso global.

Response `200`:

```json
{
    "id": "e9ef8b38-c983-4b9c-a51c-7d39ff7b1c84",
    "status": "inactive"
}
```

## Notas para frontend

Este modulo tambien aplica las reglas CRUD de `Frontend Guidelines`: crear con campos visibles/derivados, editar solo campos documentados en `Actualizar`, confirmar desactivacion y refrescar el listado despues de cada mutacion.

Este catalogo es global. No envies `X-Company-Id` para crear, actualizar o desactivar combustibles.

Campos recomendados para formularios:

- Codigo: `code`.
- Nombre: `name`.
- Estado: `status`, con `active` como valor inicial.

El `code` se normaliza a mayusculas en backend. En formularios puedes transformar visualmente a mayusculas, pero conserva la validacion del backend como fuente de verdad.

En modulos operativos, usa combustibles activos como opciones de select y muestra `name` con `code` como apoyo visual.

## Errores comunes

| Estado | Caso                                                                              |
| ------ | --------------------------------------------------------------------------------- |
| `400`  | Body, query o path params invalidos.                                              |
| `401`  | Request sin autenticacion valida.                                                 |
| `403`  | Usuario autenticado sin el permiso requerido o sin acceso global para mutaciones. |
| `404`  | Combustible no encontrado.                                                        |
| `409`  | Conflictos de unicidad segun constraints Prisma.                                  |
