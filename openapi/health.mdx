title: Health API

---

## Base

Base path: `/api/health`

Este endpoint es publico.

## Endpoint

| Metodo | Ruta          | Acceso  | Descripcion                                            |
| ------ | ------------- | ------- | ------------------------------------------------------ |
| `GET`  | `/api/health` | Publico | Verifica salud del backend y conexion a base de datos. |

## Response `200`

`GET /api/health`

```json
{
    "status": "ok",
    "checks": {
        "database": {
            "status": "ok"
        }
    }
}
```

Si la base de datos falla, el response mantiene formato y devuelve:

```json
{
    "status": "error",
    "checks": {
        "database": {
            "status": "error"
        }
    }
}
```

## Notas para frontend

Este endpoint no tiene CRUD. Aplica las reglas de `Frontend Guidelines` solo para consumo de estado: no renderices formularios, no envies headers de compania y no muestres detalles tecnicos a usuarios finales.

Este endpoint es para monitoreo, splash tecnico o verificacion de disponibilidad. No requiere autenticacion, `X-Company-Id` ni permisos.

Para UI:

- Trata `status: "ok"` como backend disponible.
- Trata `status: "error"` como degradacion de servicio.
- No muestres detalles tecnicos de infraestructura a usuarios finales; usa mensajes simples como `Servicio no disponible` y conserva el detalle para logs o panel tecnico.
