## Base

Base path: `/api/vehicles`

Todos los endpoints requieren autenticacion y permisos `vehicles.*`. El borrado desactiva el registro cambiando `status` a `inactive`.

## Alcance por compania

Requiere header `X-Company-Id: <companyId>`. Todas las operaciones quedan limitadas a esa compania. Si el acceso del usuario tiene `scopeKey: "subCompanyId"`, `subCompanyId`, `driverId` y el vehiculo consultado tambien deben pertenecer a una de esas subcompanias.

## Endpoints

| Metodo   | Ruta                       | Permiso                  | Descripcion               |
| -------- | -------------------------- | ------------------------ | ------------------------- |
| `POST`   | `/api/vehicles`            | `vehicles.create`        | Crea un vehiculo.         |
| `GET`    | `/api/vehicles`            | `vehicles.read-list`     | Lista vehiculos.          |
| `GET`    | `/api/vehicles/:id`        | `vehicles.read-one`      | Obtiene un vehiculo.      |
| `PATCH`  | `/api/vehicles/:id/driver` | `vehicles.driver.assign` | Asigna o quita conductor. |
| `PATCH`  | `/api/vehicles/:id`        | `vehicles.update`        | Actualiza un vehiculo.    |
| `DELETE` | `/api/vehicles/:id`        | `vehicles.delete`        | Desactiva un vehiculo.    |

## Listado

`GET /api/vehicles`

Query params: `page`, `limit`, `search`, `all`, `status`, `subCompanyId`, `fuelId`, `driverId`.

`search` busca por `plates`, `economicNumber` y `model`.

Para columnas visibles, usa `fuel.name`, `driver.name` y `subCompany.name`; conserva los IDs para filtros y acciones.

Response `200`: resultado paginado de `Vehicle` con `subCompany`, `fuel` y `driver` summaries.

## Crear

`POST /api/vehicles`

```json
{
    "subCompanyId": "79d03f1d-3da4-478e-806e-75ef9d86c992",
    "fuelId": "e9ef8b38-c983-4b9c-a51c-7d39ff7b1c84",
    "driverId": "d77d1344-93d4-4dc2-923a-b241185692de",
    "plates": "abc123",
    "economicNumber": "VH-001",
    "model": "Transit",
    "year": 2024,
    "odometerControl": true,
    "odometerInitial": 12500,
    "status": "active"
}
```

Requeridos: `subCompanyId`, `fuelId`, `plates`. `subCompanyId` debe estar activo y pertenecer a la compania enviada en `X-Company-Id`. `plates` se normaliza a mayusculas.

Opcionales: `driverId`, `economicNumber`, `model`, `year`, `odometerControl`, `odometerInitial`, `status`.

Response `201`: `Vehicle`.

## Obtener

`GET /api/vehicles/:id`

Path params: `id` UUID del vehiculo.

Response `200`: `Vehicle`.

## Actualizar

`PATCH /api/vehicles/:id`

Acepta `fuelId`, `driverId`, `plates`, `economicNumber`, `model`, `year`, `odometerControl`, `odometerInitial`, `status`. `driverId` puede ser `null`.

Response `200`: `Vehicle` actualizado.

## Asignar conductor

`PATCH /api/vehicles/:id/driver`

```json
{
    "driverId": "d77d1344-93d4-4dc2-923a-b241185692de"
}
```

Usa `driverId: null` para quitar conductor. El vehiculo debe estar `active` y el conductor debe estar activo en la misma sub-compania. Si se envia `X-Company-Id`, ambos deben pertenecer a esa compania.

Response:

```json
{
    "id": "db28024c-9ed7-4318-84df-eb399ebd9e7e",
    "driverId": "d77d1344-93d4-4dc2-923a-b241185692de"
}
```

## Desactivar

`DELETE /api/vehicles/:id`

Response `200`:

```json
{
    "id": "db28024c-9ed7-4318-84df-eb399ebd9e7e",
    "status": "inactive"
}
```

## Errores comunes

| Estado | Caso                                                           |
| ------ | -------------------------------------------------------------- |
| `400`  | Relacion invalida, year fuera de rango o parametros invalidos. |
| `401`  | Request sin autenticacion valida.                              |
| `403`  | Usuario autenticado sin el permiso requerido.                  |
| `404`  | Vehiculo no encontrado.                                        |
| `409`  | Conflictos de unicidad segun constraints Prisma.               |
