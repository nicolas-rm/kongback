---
title: Cards API
description: Endpoints para gestionar tarjetas y sus asignaciones a vehiculos.
---

## Base

Base path: `/api/cards`

Todos los endpoints requieren autenticacion y permisos `cards.*`. El borrado desactiva el registro cambiando `status` a `inactive`.

## Workspace

Requiere header `X-Organization-Id: <organizationId>`. Todas las operaciones quedan limitadas al workspace seleccionado. `subCompanyId`, `vehicleId` y la tarjeta consultada deben pertenecer a esa organizacion.

    },

## Endpoints

| Metodo   | Ruta                                      | Permiso                  | Descripcion                               |
| -------- | ----------------------------------------- | ------------------------ | ----------------------------------------- |
| `POST`   | `/api/cards`                              | `cards.create`           | Crea una tarjeta.                         |
| `GET`    | `/api/cards`                              | `cards.read-list`        | Lista tarjetas.                           |
| `GET`    | `/api/cards/by-design-fuel/:designFuelId` | `cards.design-fuel.read` | Lista tarjetas por combustible de diseno. |
| `GET`    | `/api/cards/:id`                          | `cards.read-one`         | Obtiene una tarjeta.                      |
| `PATCH`  | `/api/cards/:id/assign-vehicle`           | `cards.vehicle.assign`   | Asigna tarjeta a vehiculo.                |
| `PATCH`  | `/api/cards/:id/unassign`                 | `cards.unassign`         | Quita asignacion de tarjeta.              |
| `PATCH`  | `/api/cards/:id`                          | `cards.update`           | Actualiza una tarjeta.                    |
| `DELETE` | `/api/cards/:id`                          | `cards.delete`           | Desactiva una tarjeta.                    |

## Listado

`GET /api/cards`

Query params: `page`, `limit`, `search`, `all`, `status`, `subCompanyId`, `vehicleId`, `designFuelId`, `assignmentMode`.

`search` busca por `externalId`.

Para columnas visibles, usa `vehicle.plates`, `vehicle.economicNumber`, `designFuel.name` y `subCompany.name`.

Response `200`: resultado paginado de `Card`.

## Listar por combustible de diseno

`GET /api/cards/by-design-fuel/:designFuelId`

Path params: `designFuelId` UUID del combustible.

Query params: `page`, `limit`, `search`, `all`, `status`.

Valida que el combustible exista y este activo. `search` busca por `externalId`.

Response `200`: resultado paginado de `Card` filtrado por `designFuelId`.

## Crear

`POST /api/cards`

```json
{
    "subCompanyId": "79d03f1d-3da4-478e-806e-75ef9d86c992",
    "vehicleId": "db28024c-9ed7-4318-84df-eb399ebd9e7e",
    "designFuelId": "e9ef8b38-c983-4b9c-a51c-7d39ff7b1c84",
    "externalId": "CARD-001",
    "status": "active",
    "assignedAt": "2026-07-04"
}
```

Requerido: `subCompanyId`. Debe estar activo en el workspace seleccionado.

Opcionales: `vehicleId`, `designFuelId`, `externalId`, `status`, `assignedAt`. Si se envia `vehicleId`, la tarjeta queda en `assignmentMode: vehicle`; si no, queda `unassigned`.

Response `201`: `Card`.

## Obtener

`GET /api/cards/:id`

Path params: `id` UUID de la tarjeta.

Response `200`: `Card`.

## Actualizar

`PATCH /api/cards/:id`

Acepta `designFuelId`, `externalId` y `status`, todos opcionales. Si la tarjeta ya tiene vehiculo, el `designFuelId` debe coincidir con el combustible del vehiculo.

Response `200`: `Card` actualizada.

## Asignar vehiculo

`PATCH /api/cards/:id/assign-vehicle`

```json
{
    "vehicleId": "db28024c-9ed7-4318-84df-eb399ebd9e7e",
    "assignedAt": "2026-07-04"
}
```

La tarjeta y el vehiculo deben estar activos, pertenecer a la misma sub-compania y respetar `designFuelId` si existe.

Response:

```json
{
    "id": "ff2584aa-e2fc-4d75-b11c-61e447f3cbb0",
    "vehicleId": "db28024c-9ed7-4318-84df-eb399ebd9e7e",
    "assignmentMode": "vehicle",
    "assignedAt": "2026-07-04T00:00:00.000Z"
}
```

## Quitar asignacion

`PATCH /api/cards/:id/unassign`

Response:

```json
{
    "id": "ff2584aa-e2fc-4d75-b11c-61e447f3cbb0",
    "vehicleId": null,
    "assignmentMode": "unassigned",
    "assignedAt": null
}
```

## Desactivar

`DELETE /api/cards/:id`

Response `200`:

```json
{
    "id": "ff2584aa-e2fc-4d75-b11c-61e447f3cbb0",
    "status": "inactive"
}
```

## Errores comunes

| Estado | Caso                                                      |
| ------ | --------------------------------------------------------- |
| `400`  | Relacion invalida, fecha invalida o parametros invalidos. |
| `401`  | Request sin autenticacion valida.                         |
| `403`  | Usuario autenticado sin el permiso requerido.             |
| `404`  | Tarjeta no encontrada.                                    |
| `409`  | Conflictos de unicidad segun constraints Prisma.          |
