## Base

Base path: `/api/authentication`

La API soporta autenticacion con cookies HttpOnly y tambien acepta `Authorization: Bearer <accessToken>` para el access token. En login, verificacion 2FA y refresh, el backend devuelve tokens en el body y ademas setea cookies `access_token` y `refresh_token`.

Para frontend web con cookies, usa requests con credenciales incluidas. Las mutaciones autenticadas con cookies requieren `Origin` o `Referer` permitido por la proteccion CSRF. Si se usa `Authorization: Bearer`, la proteccion CSRF no aplica.

## Modelos

### TokenResponse

```json
{
    "accessToken": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
    "sessionId": "79b84132-20cc-4bf9-a4f0-3de3be656086"
}
```

| Campo          | Tipo     | Descripcion                                                                     |
| -------------- | -------- | ------------------------------------------------------------------------------- |
| `accessToken`  | `string` | JWT usado para autenticar requests. Tambien se guarda en cookie `access_token`. |
| `refreshToken` | `string` | JWT usado para renovar sesion. Tambien se guarda en cookie `refresh_token`.     |
| `sessionId`    | `string` | UUID v4 de la sesion activa.                                                    |

### TwoFactorChallengeResponse

```json
{
    "requiresTwoFactor": true,
    "challengeToken": "c8b7c6c94fa14e1ebc1a6b62d37f5e84"
}
```

```json

        {
            "id": "d32ec049-0f01-408c-b97d-c77a82d348c1",
```

### Capabilities

```json
{
    "organizationId": "2b4c918a-e7e0-4e99-b6c1-c2efed00b478",
    "isGlobalAdmin": false,
    "permissions": ["cards.read-list", "vehicles.read-list"]
}
```

| Campo            | Tipo       | Descripcion                                                                 |
| ---------------- | ---------- | --------------------------------------------------------------------------- |
| `organizationId` | `string`   | UUID v4 del workspace usado para calcular permisos efectivos.               |
| `isGlobalAdmin`  | `boolean`  | Indica si el usuario tiene acceso admin global.                             |
| `permissions`    | `string[]` | Codigos de permisos efectivos para el workspace seleccionado, sin duplicar. |

## Headers utiles

| Header                                | Uso                                                                                                                                  |
| ------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| `Authorization: Bearer <accessToken>` | Alternativa a cookie `access_token` para endpoints autenticados.                                                                     |
| `X-Organization-Id`                   | UUID del workspace seleccionado. Requerido por endpoints tenant-scoped como companias, vehiculos, tarjetas, estaciones y documentos. |
| `X-Company-Id`                        | UUID opcional de la compania seleccionada dentro del workspace. Si se envia, el backend valida que pertenezca a `X-Organization-Id`. |
| `Accept-Language`                     | Idioma para validaciones, errores y correos cuando aplica.                                                                           |
| `X-Device-Name`                       | Nombre del dispositivo guardado al crear sesion con login o login 2FA.                                                               |
| `X-Forwarded-For`                     | IP de cliente usada para contexto de sesion si existe.                                                                               |
| `X-Request-Id`                        | ID de request opcional; el backend lo devuelve como response header.                                                                 |

## Login

`POST /api/authentication/login`

Publico. Rate limit: 5 requests por 60 segundos.

### Body

```json
{
    "username": "admin",
    "password": "Secret123"
}
```

| Campo      | Tipo     | Requerido | Reglas                         |
| ---------- | -------- | --------- | ------------------------------ |
| `username` | `string` | Si        | Se recorta y colapsa espacios. |
| `password` | `string` | Si        | Se recorta y colapsa espacios. |

### Response `200`

Si el usuario no tiene 2FA activo, devuelve `TokenResponse` y setea cookies `access_token` y `refresh_token`.

Si el usuario tiene 2FA activo, limpia cookies existentes y devuelve `TwoFactorChallengeResponse`.

## Verificar login 2FA

`POST /api/authentication/2fa/verify-login`

Publico. Rate limit: 5 requests por 60 segundos.

### Body

Con codigo TOTP:

```json
{
    "challengeToken": "c8b7c6c94fa14e1ebc1a6b62d37f5e84",
    "code": "123456"
}
```

Con recovery code:

```json
{
    "challengeToken": "c8b7c6c94fa14e1ebc1a6b62d37f5e84",
    "recoveryCode": "ABCD-EFGH"
}
```

| Campo            | Tipo     | Requerido   | Reglas                                                                                         |
| ---------------- | -------- | ----------- | ---------------------------------------------------------------------------------------------- |
| `challengeToken` | `string` | Si          | Token devuelto por `POST /api/authentication/login` cuando `requiresTwoFactor` es `true`.      |
| `code`           | `string` | Condicional | Codigo TOTP. Requerido si no se envia `recoveryCode`.                                          |
| `recoveryCode`   | `string` | Condicional | Recovery code de un solo uso. Requerido si no se envia `code`. Acepta formato con o sin guion. |

### Response `200`

Devuelve `TokenResponse` y setea cookies `access_token` y `refresh_token`. Si se usa `recoveryCode`, el backend lo marca como usado y no puede reutilizarse.

## Registro

`POST /api/authentication/register`

Publico. Rate limit: 3 requests por 60 segundos.

### Body

```json
{
    "username": "admin",
    "email": "admin@example.com",
    "fullName": "Admin User",
    "password": "Secret123"
}
```

| Campo      | Tipo     | Requerido | Reglas                                                         |
| ---------- | -------- | --------- | -------------------------------------------------------------- |
| `username` | `string` | Si        | Se recorta y colapsa espacios.                                 |
| `email`    | `string` | Si        | Se recorta, colapsa espacios y normaliza a minusculas.         |
| `fullName` | `string` | Si        | Se recorta y colapsa espacios.                                 |
| `password` | `string` | Si        | Minimo 8 caracteres, una minuscula, una mayuscula y un numero. |

### Response `200`

```json
{
    "id": "9f5f0f4a-8f88-4f0b-9a18-6eb3f1e9b7a1",
    "registered": true,
    "emailVerificationRequired": true
}
```

## Refresh

`POST /api/authentication/refresh`

Publico. Rate limit: 20 requests por 60 segundos.

El refresh token puede enviarse en el body o venir desde la cookie `refresh_token`. El backend revoca el refresh token usado, actualiza actividad de sesion y emite nuevos tokens.

### Body

```json
{
    "refreshToken": "eyJhbGciOiJIUzI1NiIs..."
}
```

| Campo          | Tipo     | Requerido | Reglas                                     |
| -------------- | -------- | --------- | ------------------------------------------ |
| `refreshToken` | `string` | No        | Opcional si existe cookie `refresh_token`. |

### Response `200`

Devuelve `TokenResponse` y actualiza cookies `access_token` y `refresh_token`.

## Logout

`POST /api/authentication/logout`

Autenticado.

Limpia cookies de autenticacion. Si el request trae cookie `refresh_token`, revoca la sesion asociada a ese refresh token. Si no trae refresh cookie, el comportamiento actual del servicio revoca todas las sesiones del usuario autenticado.

### Response `200`

```json
{
    "revokedSessions": 1
}
```

## Logout all

`POST /api/authentication/logout-all`

Autenticado.

Revoca todas las sesiones del usuario y limpia cookies.

### Response `200`

```json
{
    "revokedSessions": 3
}
```

## Perfil actual

`GET /api/authentication/me`

Autenticado. Este endpoint se puede usar aunque `mustChangePassword` sea `true`.

### Response `200`

Devuelve `Profile`. Este response no incluye permisos; para decisiones de UI por workspace, usa `GET /api/authentication/capabilities`.

## Listar workspaces

`GET /api/authentication/workspaces`

Autenticado. Este endpoint se puede usar aunque `mustChangePassword` sea `true`.

Devuelve los workspaces que el usuario puede seleccionar en el front. Un admin global recibe todas las organizaciones activas con `isGlobalAdmin: true`; un usuario normal recibe solo las organizaciones donde tiene accesos activos.

### Response `200`

Devuelve `Workspace[]`.

## Consultar capacidades

`GET /api/authentication/capabilities`

Autenticado. Este endpoint se puede usar aunque `mustChangePassword` sea `true`.

Requiere header `X-Organization-Id: <organizationId>`. El backend valida que el workspace exista, este activo y sea accesible para el usuario antes de calcular permisos.

Este endpoint es el contrato recomendado para renderizar sidebar, modulos y acciones dependientes del cliente/workspace seleccionado.

### Response `200`

Devuelve `Capabilities`.

### Path params

| Parametro   | Tipo   | Descripcion                |

### Response `200`

## Cambiar contrasena

`PATCH /api/authentication/password`

Autenticado. Rate limit: 5 requests por 60 segundos. Este endpoint se puede usar aunque `mustChangePassword` sea `true`.

Al cambiar la contrasena, el backend revoca sesiones del usuario y limpia cookies. El frontend debe enviar al usuario a login al recibir respuesta exitosa.

### Body

```json
{
    "currentPassword": "Secret123",
    "newPassword": "NewSecret123"
}
```

| Campo             | Tipo     | Requerido | Reglas                                                         |
| ----------------- | -------- | --------- | -------------------------------------------------------------- |
| `currentPassword` | `string` | Si        | Contrasena actual.                                             |
| `newPassword`     | `string` | Si        | Minimo 8 caracteres, una minuscula, una mayuscula y un numero. |

### Response `200`

```json
{
    "passwordChanged": true
}
```

## Iniciar setup 2FA

`POST /api/authentication/2fa/setup`

Autenticado. Rate limit: 3 requests por 60 segundos. Este endpoint se puede usar aunque `mustChangePassword` sea `true`.

Genera un secreto TOTP pendiente y una URL `otpauth://` para QR.

### Response `200`

```json
{
    "secret": "JBSWY3DPEHPK3PXP",
    "otpAuthenticationUrl": "otpauth://totp/Kong:admin%40example.com?secret=JBSWY3DPEHPK3PXP&issuer=Kong"
}
```

## Flujo recomendado para frontend

1. Llamar `POST /api/authentication/login`.
2. Si responde `requiresTwoFactor: true`, pedir codigo TOTP y llamar `POST /api/authentication/2fa/verify-login`.
3. Guardar estado de sesion desde cookies o desde `TokenResponse`, segun estrategia del cliente.
4. Llamar `GET /api/authentication/me` para obtener identidad, `organizationIds` y `mustChangePassword`.
5. Llamar `GET /api/authentication/workspaces`, permitir que el usuario seleccione workspace y enviar su `id` como `X-Organization-Id` en endpoints tenant-scoped.
6. Llamar `GET /api/authentication/capabilities` con `X-Organization-Id` para renderizar modulos y acciones disponibles en el workspace seleccionado.
7. Opcionalmente seleccionar compania dentro del workspace y enviar su `id` como `X-Company-Id` para trabajar solo con datos de esa compania.
8. Si `mustChangePassword` es `true`, limitar la UI a cambio de contrasena, logout, ver perfil, workspaces y endpoints marcados como permitidos arriba.
9. Renovar sesion con `POST /api/authentication/refresh` cuando expire el access token.

## Errores comunes

| Estado | Caso                                                                                                         |
| ------ | ------------------------------------------------------------------------------------------------------------ |
| `400`  | Body invalido, token de reset/verificacion invalido, codigo 2FA invalido o configuracion 2FA expirada.       |
| `401`  | Credenciales invalidas, access token ausente/invalido/expirado, refresh token invalido/expirado/reutilizado. |
| `403`  | CSRF rechazado o usuario con `mustChangePassword` intentando usar endpoints bloqueados.                      |
| `409`  | Conflictos de unicidad como `username` o `email` duplicados, segun el filtro Prisma global.                  |
| `429`  | Rate limit excedido en endpoints protegidos con throttling.                                                  |
